# Cocos Creator 精灵图切分问题解决方案

## 问题分析

脚本已成功修改 `.meta` 文件，但 Cocos Creator 未识别更改。

原因：Cocos Creator 3.x 有编译缓存，需要清理后重新导入。

---

## 解决方案

### 方案 1：清理缓存并重启编辑器（推荐）

1. **完全关闭 Cocos Creator**
2. **删除缓存目录**：
   ```
   删除项目根目录下的 library/ 文件夹
   删除项目根目录下的 temp/ 文件夹（如果存在）
   ```
3. **重新打开项目**
4. 等待资源重新导入完成

### 方案 2：使用编辑器重新导入

在 Cocos Creator 中：
1. 选中所有精灵图文件（Ctrl+A 选中 characters 文件夹下的所有图片）
2. 右键 → **重新导入资源**
3. 等待导入完成

### 方案 3：手动触发刷新

1. 将一张精灵图移出 characters 文件夹
2. 等待编辑器识别（资源消失）
3. 再将图片移回 characters 文件夹
4. 编辑器会重新读取 .meta 配置

---

## 下一步操作

**推荐执行方案 1**，步骤：
1. 保存所有场景和文件
2. 完全退出 Cocos Creator
3. 删除项目中的 `library/` 文件夹
4. 重新打开项目
5. 等待资源重新导入（约1-2分钟）

---

删除 library 文件夹是安全的，它只是编译缓存，会自动重新生成。
